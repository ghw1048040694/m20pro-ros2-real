# M20Pro 跨楼层导航方案

## 成功定义

跨楼层成功不是前端切换了地图，也不是 `/m20pro/current_floor` 字符串发生变化。只有以下三项同时确认，系统才进入目标楼层并继续原任务：

1. 106 已把目标层原厂地图包 `drmap apply` 为 active。
2. 104 Nav2 `/map` 已确认与目标层归档地图一致。
3. 目标层 2101 重定位已返回本次新鲜确认，定位、位姿和激光证据通过。

正式执行链路为：

```text
任务点 -> 楼梯入口 -> 三维净空连续确认 -> 楼梯步态 -> 起始层共享平台
-> 106/104 地图原子切换 -> 目标层 2101 重定位
-> 出口三维净空确认 -> 目标层出口 -> 平地步态 -> 继续目标层任务点
```

## 建图和标定

1. 在前端“建图”选择多楼层，填写项目、建筑/区域、地图名称和现场真实楼层，例如 `F1,F2`。
2. 点击“启动 106 建图”。系统自动检查 106、创建或复用当前多楼层会话并开始当前层建图。
3. 点击“完成/保存建图”，再“从 106 拉到 104”。确认导入地图的实际楼层正确。
4. 在建图步骤中切换下一层，再次启动、完成和导入。切换步骤不会新建会话或跳回第一层。
5. 分别选择每层正式地图并完成重定位。在每个方向标定四点：

| 点 | 所属地图 | 类型 | 含义 |
| --- | --- | --- | --- |
| 起始层入口 | 起始层 | 爬楼梯点 | 平地导航结束、准备切楼梯步态的位置 |
| 起始层切换点 | 起始层 | 楼层切换点 | 以起始层坐标到达共享平台的位置 |
| 目标层切换点 | 目标层 | 楼层切换点 | 同一物理平台在目标层地图中的定位坐标 |
| 目标层出口 | 目标层 | 出楼梯点 | 离开楼梯区域、恢复平地步态的位置 |

四点必须来自各自正式地图。切换点不是随便取的中间点，两侧切换点应描述同一安全物理平台在两张地图中的坐标。

## 路线配置

1. 在“建图 -> 跨楼层路线”依次选择四点并保存。
2. 路线是有向的。需要上楼和下楼时，必须分别标定并保存 `F1 -> F2` 与 `F2 -> F1`。
3. 同一楼层的所有路线必须使用同一张正式地图，避免 104 和 106 对同一楼层产生多个地图真值。
4. 地图或点位被路线引用后不能直接改删；先删除路线，重新标定后再保存。

## 任务执行

1. 先选择顺序中首个任务点所在地图并完成重定位。
2. 在“任务 -> 跨楼层编排”按真实执行顺序加入点位。例如要先去 F1-A，再去 F2-C，最后回 F1-B，就按该顺序加入。
3. 确认“楼层路径”预览没有“缺少路线”，再生成跨楼层任务。
4. 开始任务后查看顶部任务状态。楼梯阶段会依次显示前往入口、通过平台、同步切图和定位、离开楼梯、继续任务点。
5. 首次测试只做相邻楼层单程；确认成功后测试反向路线，最后再做多点和多层组合。

## 楼梯避障与安全合同

平地和楼梯不能共用同一种二维障碍解释。106 始终保留普通 `/scan`，但在楼梯安全会话中额外使用原始三维点云建立前方踏面包络。默认配置把相邻高度不超过 `0.24 m` 的连续台阶视为可通行地形，把高于局部踏面 `0.26 m` 的稳定点簇或超过单级台阶高度的突变视为异常障碍；实际值以唯一现场配置为准。104 的 Nav2 障碍层统一读取 `/m20pro/navigation_scan`；平地时该话题来自普通 `/scan`，楼梯时来自只保留异常障碍的 `/m20pro/stair_obstacle_scan`，AMCL、前端和录包仍保留原始 `/scan`。

楼梯起步和目标层出口前都要求至少三个连续 `clear` 样本。运动中收到 `blocked`、`unknown`、会话不匹配、数据超过 1 秒未更新或 1.5 秒模式心跳租约失效，`floor_manager` 会发布零速度并取消 Nav2。楼梯专用行为树只有规划与跟踪，不允许 `BackUp`、`Spin`、清空代价地图或其他运动恢复；检测到人或箱体后只能停止，由现场清障并重新开始任务，不能在台阶上自主绕行。

现场验收必须同时覆盖空楼梯、上行、下行、高物体阻塞、人员临时遮挡和感知断流。雷达安装高度、俯仰角或机器狗楼梯姿态变化后必须重新录包校准参数。低于单级台阶高度且几何上与踏面连续的小物体无法只靠单一三维几何可靠区分，楼梯在执行前仍必须人工清场；该边界不能通过降低阈值规避，否则会重新把台阶本身判成障碍。

## 现场参数

点云高度带、楼梯几何阈值、净空样本/超时、楼层切换等待、Nav2 运动/规划/代价地图和定位稳定性参数统一写在 `src/m20pro_bringup/config/m20pro_field_profile.yaml`。schema v2 共开放 67 个现场参数，其中导航 28 个。`stair.obstacle_height_m` 不可单独配置，而是由 `max_step_height_m + obstacle_height_margin_m` 生成，避免台阶上限和障碍阈值分叉。

上位机先运行 `./scripts/apply_field_profile.sh --check`，确认后再运行 `./scripts/apply_field_profile.sh`。后者只允许在无活动任务时更新，完整部署 106 和 104 后核对两端配置哈希；任何一端不一致都不能进入楼梯模式。不要编辑 106 `/etc` 生成文件，不支持任务中热更新，也不保留旧值回退。

提高 `max_step_height_m` 只会改变感知分类，不能提高关节、步态、附着力或整机通过能力。工地临时高台阶必须先获得厂家机械能力边界，再以空载、有人看护、手柄急停和录包的方式逐级验证。

## 失败边界

- 当前地图不是路线起始地图、任务不是跨楼层任务、请求路线字段不一致时，切图前直接拒绝。
- 104 或 106 任一地图切换失败时，系统回滚两端地图并停止任务。
- 目标层重定位未确认或切层期间任务被停止时，系统回滚到起始地图并停止任务。
- 楼梯净空未连续确认时不切步态、不下发楼梯目标；运动中净空受阻、未知、丢失或过期时立即停止，不执行倒退、原地旋转或绕行动作。
- 回滚无法完整确认时，当前楼层置为空，必须人工选择正确地图并重新定位后才能继续。
- 现场不使用 F19/F20/F21 历史样例代替真实地图，不根据楼层编号自动生成路线，不用地图显示变化代替成功证据。
