# M20Pro 跨楼层导航方案

> 当前运行状态（2026-07-22）：保留多层建图、楼层地图、四点路线和跨层任务编排数据，但旧楼梯感知、扫描选择器和爬楼执行链已撤除。新爬楼方案接入前，系统会在跨层运动前返回 `stair_execution_retired`，禁止真实跨层测试。下文的切图事务和路线定义供新方案复用。

## 成功定义

跨楼层成功不是前端切换了地图，也不是 `/m20pro/current_floor` 字符串发生变化。只有以下三项同时确认，系统才进入目标楼层并继续原任务：

1. 106 已把目标层原厂地图包 `drmap apply` 为 active。
2. 104 Nav2 `/map` 已确认与目标层归档地图一致。
3. 目标层 2101 重定位已返回本次新鲜确认，定位、位姿和激光证据通过。

新方案将复用的跨层事务边界为：

```text
任务点 -> 新爬楼执行器 -> 起始层共享平台
-> 106/104 地图原子切换 -> 目标层 2101 重定位
-> 新爬楼执行器完成退出 -> 继续目标层任务点
```

## 建图和标定

1. 在前端“建图”选择多楼层，填写项目、建筑/区域、地图名称和现场真实楼层，例如 `F1,F2`。
2. 点击“启动 106 建图”。系统自动检查 106、创建或复用当前多楼层会话并开始当前层建图。
3. 点击“完成/保存建图”，再“从 106 拉到 104”。确认导入地图的实际楼层正确。
4. 在建图步骤中切换下一层，再次启动、完成和导入。切换步骤不会新建会话或跳回第一层。
5. 分别选择每层正式地图并完成重定位。在每个方向标定四点：

| 点 | 所属地图 | 类型 | 含义 |
| --- | --- | --- | --- |
| 起始层入口 | 起始层 | 爬楼梯点 | 平地导航结束、准备切楼梯步态的位置 |
| 起始层切换点 | 起始层 | 楼层切换点 | 以起始层坐标到达共享平台的位置 |
| 目标层切换点 | 目标层 | 楼层切换点 | 同一物理平台在目标层地图中的定位坐标 |
| 目标层出口 | 目标层 | 出楼梯点 | 离开楼梯区域、恢复平地步态的位置 |

四点必须来自各自正式地图。切换点不是随便取的中间点，两侧切换点应描述同一安全物理平台在两张地图中的坐标。

## 路线配置

1. 在“建图 -> 跨楼层路线”依次选择四点并保存。
2. 路线是有向的。需要上楼和下楼时，必须分别标定并保存 `F1 -> F2` 与 `F2 -> F1`。
3. 同一楼层的所有路线必须使用同一张正式地图，避免 104 和 106 对同一楼层产生多个地图真值。
4. 地图或点位被路线引用后不能直接改删；先删除路线，重新标定后再保存。

每条有向路线同时绑定 106 本地 `terrain_guard` 的 `profile_id` 和
`corridor_version`。新路线默认为 `shadow-v1`、`stop_only`，只用于影子感知
和版本校验；未经录包、现场楼梯和执行器验收，不能把它解释为允许运动。

## 任务编排（执行暂停）

1. 先选择顺序中首个任务点所在地图并完成重定位。
2. 在“任务 -> 跨楼层编排”按真实执行顺序加入点位。例如要先去 F1-A，再去 F2-C，最后回 F1-B，就按该顺序加入。
3. 确认“楼层路径”预览没有“缺少路线”，再生成跨楼层任务。
4. 当前可校验路线预览和任务顺序，不启动真实跨层运动。误启动时应显示旧爬楼链路已停用。

## 平地感知合同

106 只发布完整滚动聚合 `/scan`；104 的 Nav2 局部/全局代价地图、前端、自检和录包都直接消费这一话题。不再存在楼梯模式、二次扫描选择或障碍裁剪链路。

楼梯连接边使用独立的语义执行状态机：入口和目标层出口仍由楼层内 Nav2
负责，楼梯本体只允许认证连接器沿三维走廊通过；遇到人员、箱体、点云过期或
定位异常立即停止，不在台阶上自由绕行。当前状态机只有合同和离线测试，尚未
接入真实速度输出，`stair_execution_retired` 继续有效。

## 现场参数

点云高度带、楼层切换等待、Nav2 运动/规划/代价地图、网页遥控安全边界和定位稳定性参数统一写在 `src/m20pro_bringup/config/m20pro_field_profile.yaml`。schema v4 共开放 66 个现场参数，其中导航 29 个、遥控 7 个；旧楼梯感知配置组已删除。

上位机先运行 `./scripts/apply_field_profile.sh --check`，确认后再运行 `./scripts/apply_field_profile.sh`。后者只允许在无活动任务时更新，完整部署 106 和 104 后核对两端配置哈希。不要编辑 106 `/etc` 生成文件，不支持任务中热更新，也不保留旧值回退。

## 失败边界

- 当前地图不是路线起始地图、任务不是跨楼层任务、请求路线字段不一致时，切图前直接拒绝。
- 104 或 106 任一地图切换失败时，系统回滚两端地图并停止任务。
- 目标层重定位未确认或切层期间任务被停止时，系统回滚到起始地图并停止任务。
- 当前任何跨层运动请求都在起步前以 `stair_execution_retired` 拒绝。
- 回滚无法完整确认时，当前楼层置为空，必须人工选择正确地图并重新定位后才能继续。
- 现场不使用 F19/F20/F21 历史样例代替真实地图，不根据楼层编号自动生成路线，不用地图显示变化代替成功证据。
