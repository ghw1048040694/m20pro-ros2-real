# M20Pro 跨楼层巡检导航

## 当前链路

系统只保留一套统一导航。单层任务是只有一个楼层段的统一任务，跨层任务只是在相邻楼层段之间多一条有向楼梯连接边。

```text
统一任务
-> stair_executor
-> Nav2 到楼梯入口
-> /m20pro/gait_command 切换楼梯步态
-> /cmd_vel_nav -> command_mux -> 103 运动链
-> 到达 source_platform 并停车
-> 现有 104/106 切图事务与 2101 重定位
-> Nav2 离开楼梯
-> 恢复 flat 步态
-> 继续原任务
```

上楼使用正向低速，下楼按 X30Pro 原厂操作要求使用后退低速。速度仍经过现有 `command_mux`，不新增绕过仲裁直发 103 的通道。

旧 `floor_manager.use_stairs()` 只保留明确拒绝，防止历史入口与新链并行。已删除第二个动作编排节点。`terrain_guard` 当前只可在 106 做影子观测，不作为首轮跨层运动的前置门禁，也不改变平地 `/scan`。

## 成功定义

一条跨层任务只有在以下顺序全部完成后才算成功：

1. Nav2 到达源层楼梯入口。
2. 机器狗切换正确楼梯步态并到达共享平台。
3. 106 已 apply 目标层原厂地图，104 Nav2 已切换对应地图。
4. 目标层 `target_platform` 的 2101 重定位成功。
5. Nav2 到达目标层出口，恢复平地步态并继续任务点。

仅前端楼层文字发生变化不算切层成功。

## 建图与路线

1. 在前端建图页填写项目、建筑/区域、地图名称和现场真实楼层。楼层不限于 `F19/F20/F21`。
2. 每层分别执行“启动 106 建图”、“完成/保存建图”和“拉取 106 建图结果”。拉取只归档，不代替地图切换和重定位。
3. 在两层正式地图上分别完成重定位，然后标定一条有向路线的四个点。

| 点位 | 所属地图 | 用途 |
| --- | --- | --- |
| `entry` | 源层 | Nav2 结束平地导航、准备切步态 |
| `source_platform` | 源层 | 楼梯直行的停车目标与切图位置 |
| `target_platform` | 目标层 | 切图后 2101 重定位位姿 |
| `post_exit` | 目标层 | Nav2 离开楼梯后恢复平地步态的目标 |

`source_platform` 和 `target_platform` 描述同一个物理共享平台在两张地图中的坐标。上楼和下楼是两条有向路线，必须分别实测和保存，不由系统自动镜像生成。

## 任务执行

1. 选择首个任务点所在的地图并完成重定位。
2. 在任务页按真实顺序加入任务点。点位顺序就是执行顺序，系统会在连续楼层段之间解析已保存的有向路线。
3. 确认路线预览没有“缺少路线”，然后生成并执行统一任务。
4. 真实运动必须使用 `104_start_real_move.sh`。`104_start_real_shadow.sh` 不启动楼梯执行器，不用于跨层运动验收。

## 首轮必要判定

为先跑通最小闭环，起步前只检查：

- 有向路线具有路线、源/目标地图和源/目标楼层身份；
- 方向是 `up` 或 `down`；
- `entry/source_platform/target_platform/post_exit` 四个位姿完整；
- 唯一 `stair_executor` 已启动且没有执行上一条路线。

运动中只保留直接影响闭环的停止条件：

- 用户停止任务；
- 定位明确失效；
- 机器狗位姿超过 `motion_pose_timeout_s` 未更新；
- 当前阶段超过 `connector_stage_timeout_s`；
- 104/106 切图或 2101 重定位失败。

首轮不要求 terrain 状态、走廊认证、另一个动作编排器或多层运动 readiness 同时通过。后续只根据真机实测出现的明确风险增加加固项。

## 现场参数

只在 `src/m20pro_bringup/config/m20pro_field_profile.yaml` 修改现场参数。楼梯首次测试的核心项为：

```yaml
stair_transition:
  motion_speed_mps: 0.12
  motion_command_hz: 10.0
  motion_pose_timeout_s: 1.50
  gait_settle_s: 1.0
  connector_stage_timeout_s: 180.0
  platform_position_tolerance_m: 0.50
```

修改后执行：

```bash
./scripts/apply_field_profile.sh --check
./scripts/apply_field_profile.sh
```

## 最小现场验收

1. 只选择一段规则、空间充足、中间有可停车平台的固定楼梯。
2. 只配置两层、一条上行路线、每层一个近距离任务点。
3. 先分别验证两张地图的人工切换和重定位，再执行上行单程。
4. 现场人员在机器狗侧后方看护，原厂手柄急停随时可用，全程录包。
5. 上行完整通过后，再单独配置和验证反向下楼路线。
6. 首次验收不同时叠加雷达、YOLO、长距离多点和复杂巡检。

当前分支已完成离线合同测试和启动接线，但尚未在真实楼梯完成首轮运动验收，因此不应宣称已可无人跨层运行。
